    Controller PVs¶

    Controller-level PVs provide identity information, status monitoring, and global configuration for a Galil controller. They are defined in two templates:

    • galil_dmc_ctrl.template -- for DMC motion controllers (full feature set)
    • galil_rio_ctrl.template -- for RIO remote I/O controllers (subset)

    All PVs use the prefix $(P), which is typically set to something like DMC01: or RIO01: in the IOC startup script.

    DMC Controller PVs¶

    The DMC template exposes the complete set of controller PVs described in the sections below.

    Controller Identity and Status¶

    These read-only PVs report the controller's hardware identity and communication health.

    PV Suffix Record Type Direction asyn drvInfo Description
    DRIVER_MON stringin Read CONTROLLER_DRIVER Driver version string
    MODEL_MON stringin Read CONTROLLER_MODEL Controller model (e.g. "DMC4143"). Galil ID command
    ADDRESS_MON stringin Read CONTROLLER_ADDRESS Controller address (IP or serial port)
    ETHADDR_MON stringin Read CONTROLLER_ETHADDR Ethernet MAC address
    SERIALNUM_MON stringin Read CONTROLLER_SERIALNUM Controller serial number
    COMMERR_STATUS bi Read CONTROLLER_COMMERR Communication error. 0 = OK, 1 = Error (MAJOR alarm)
    START_STATUS bi Read CONTROLLER_START Startup status. 0 = Error (MAJOR alarm), 1 = OK
    SSICAPABLE_STATUS bi Read CONTROLLER_SSICAPABLE SSI encoder capable (based on firmware/hardware)
    BISSCAPABLE_STATUS bi Read CONTROLLER_BISSCAPABLE BiSS encoder capable
    PVTCAPABLE_STATUS bi Read CONTROLLER_PVTCAPABLE PVT trajectory capable
    ECATCAPABLE_STATUS bi Read CONTROLLER_ECATCAPABLE EtherCat capable

    Switch Configuration¶

    These PVs configure how the controller interprets limit and home switches. They map to the Galil CN (Configure) command.

    PV Suffix Record Type Direction asyn drvInfo Description
    LIMITTYPE_CMD bo Write CONTROLLER_LIMITTYPE Set limit switch type: NO (normally open) or NC (normally closed)
    LIMITTYPE_STATUS bi Read CONTROLLER_LIMITTYPE Readback of limit switch type
    HOMETYPE_CMD bo Write CONTROLLER_HOMETYPE Set home switch type: NO or NC
    HOMETYPE_STATUS bi Read CONTROLLER_HOMETYPE Readback of home switch type
    HOMEEDGE_CMD bo Write CONTROLLER_HOMEEDGE Home edge: Reverse or Forward
    HOMEEDGE_STATUS bi Read CONTROLLER_HOMEEDGE Readback of home edge

    EtherCat¶

    These PVs control and monitor the EtherCat network interface, available on controllers with EtherCat hardware.

    PV Suffix Record Type Direction asyn drvInfo Description
    ECATNETWORK_CMD bo Write CONTROLLER_ECAT_NETWORK Enable or disable the EtherCat network
    ECATNETWORK_STATUS bi Read CONTROLLER_ECAT_NETWORK EtherCat network status
    ECATFLT_STATUS ai Read CONTROLLER_ECAT_FAULT EtherCat fault code (polled at 1 second)

    Deferred Moves¶

    Deferred moves allow multiple axes to start simultaneously. When deferred mode is active, the Galil BG (Begin) command is held until the mode is set back to Go.

    PV Suffix Record Type Direction asyn drvInfo Description
    DEFER_CMD bo Write MOTOR_DEFER_MOVES Set deferred mode: Go (execute immediately) or Defer (buffer moves). Issues the BG command when set to Go
    DEFER_STATUS bi Read MOTOR_DEFER_MOVES Deferred moves status
    DEFER_MODE_CMD bo Write CONTROLLER_DEFERRED_MODE Deferred mode: "Sync start only" or "Sync start/stop". When set to "Sync start/stop", the ST (Stop) command is also deferred
    DEFER_MODE_STATUS bi Read CONTROLLER_DEFERRED_MODE Deferred mode readback

    Coordinate System Selection¶

    PV Suffix Record Type Direction asyn drvInfo Description
    COORDSYS_CMD bo Write COORDINATE_SYSTEM Select active coordinate system: S or T. Galil CS (Clear Sequence) command
    COORDSYS_STATUS bi Read COORDINATE_SYSTEM Active coordinate system readback

    Output Compare¶

    Output compare generates a pulse on a digital output when the encoder crosses specified positions. It uses the Galil OC (Output Compare) command. Two output compare channels are available: OC1 covers axes A through D, and OC2 covers axes E through H.

    PV Suffix Record Type Direction asyn drvInfo Description
    OC1START_CMD ao Write OUTPUT_COMPARE_START (addr 0) Output compare 1 start position
    OC1START_MON ai Read OUTPUT_COMPARE_START (addr 0) Readback
    OC1INCR_CMD ao Write OUTPUT_COMPARE_INCR (addr 0) Output compare 1 increment
    OC1INCR_MON ai Read OUTPUT_COMPARE_INCR (addr 0) Readback
    OC1AXIS_CMD mbbo Write OUTPUT_COMPARE_AXIS (addr 0) Output compare 1 axis: OFF, A, B, C, D
    OC1AXIS_STATUS mbbi Read OUTPUT_COMPARE_AXIS (addr 0) Readback
    OC2START_CMD ao Write OUTPUT_COMPARE_START (addr 1) Output compare 2 start position
    OC2START_MON ai Read OUTPUT_COMPARE_START (addr 1) Readback
    OC2INCR_CMD ao Write OUTPUT_COMPARE_INCR (addr 1) Output compare 2 increment
    OC2INCR_MON ai Read OUTPUT_COMPARE_INCR (addr 1) Readback
    OC2AXIS_CMD mbbo Write OUTPUT_COMPARE_AXIS (addr 1) Output compare 2 axis: OFF, E, F, G, H
    OC2AXIS_STATUS mbbi Read OUTPUT_COMPARE_AXIS (addr 1) Readback
    OCMESSAGE_MON waveform Read OUTPUT_COMPARE_MESSAGE Output compare status message

    User Array Upload¶

    PV Suffix Record Type Direction asyn drvInfo Description
    UPLOAD_CMD bo Write CONTROLLER_UARRAY_UPLOAD Trigger upload of user arrays from controller. Galil QU (Upload Array) command
    UPLOAD_STATUS bi Read CONTROLLER_UARRAY_UPLOAD Upload status: Idle or Uploading

    Error Display¶

    PV Suffix Record Type Direction asyn drvInfo Description
    ERROR_MON waveform Read CONTROLLER_ERROR Controller error message (256-character waveform)

    Command Console¶

    These PVs provide a pass-through interface for sending arbitrary Galil commands and reading the responses.

    PV Suffix Record Type Direction asyn drvInfo Description
    SEND_STR_CMD stringout Write USER_OCTET Send any Galil command string to the controller
    SEND_STR_MON waveform Read USER_OCTET Response string from the last command (256-character waveform)
    SEND_STRVAL_MON ai Read USER_OCTET_VAL Numeric value parsed from the last command response

    Amplifier Fault Clear¶

    PV Suffix Record Type Direction asyn drvInfo Description
    CLEARAMPFAULTS_CMD bo Write CONTROLLER_CLEARAMPFAULTS Clear amplifier faults. Galil AZ (Clear Latched Amplifier Errors) command

    RIO Controller PVs¶

    The RIO template (galil_rio_ctrl.template) provides a subset of the DMC controller PVs, limited to what is relevant for a remote I/O controller without motor axes.

    Included PVs¶

    Identity: DRIVER_MON, MODEL_MON, ADDRESS_MON, ETHADDR_MON, SERIALNUM_MON

    Status: COMMERR_STATUS, START_STATUS

    User Array Upload: UPLOAD_CMD, UPLOAD_STATUS

    Error Display: ERROR_MON

    Command Console: SEND_STR_CMD, SEND_STR_MON, SEND_STRVAL_MON

    These PVs have the same record types, asyn drvInfo strings, and behavior as their DMC counterparts described above.

    Features Not Available on RIO¶

    The following DMC features are not present in the RIO template:

    • Switch configuration (LIMITTYPE, HOMETYPE, HOMEEDGE)
    • Deferred moves (DEFER, DEFER_MODE)
    • Coordinate system selection (COORDSYS)
    • Output compare (OC1*, OC2*, OCMESSAGE)
    • EtherCat (ECATNETWORK, ECATFLT)
    • Capability flags (SSICAPABLE, BISSCAPABLE, PVTCAPABLE, ECATCAPABLE)
    • Amplifier fault clear (CLEARAMPFAULTS)
